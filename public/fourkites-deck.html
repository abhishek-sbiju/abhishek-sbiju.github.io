12 / THE ORACLEDUAL-ENGINE RECONCILIATION

A green that shouldn't be green
cannot ship

The failure mode it kills

Migrating tests at volume produces greens that assert nothing. The oracle's entire job is to catch them — by machinery, not by hope.

How it works

Every migrated journey is run twice: the new BDD test on QAT, and the original Virtuoso recording. The verdicts are reconciled checkpoint by checkpoint.

Proof it's real

The “both fail” branch found a genuine app defect — filed as a defect ticket instead of papering over it.

VIRTUOSO SAYS ✅VIRTUOSO SAYS ❌
BDD ✅ · VIRTUOSO ✅

Proceed

Both engines agree the behavior passes — the migration is faithful. Ship it.

BDD ✅ · VIRTUOSO ❌

🚨 False green — hard-block

The migrated test passes where the recording fails. The lane refuses to let it merge.

BDD ❌ · VIRTUOSO ✅

Heal & re-run

The migration is wrong, not the app — back into the heal loop it goes.

BDD ❌ · VIRTUOSO ❌

Real app defect — park

Both engines fail: the app is broken. Park the journey, remove the files, file a defect.

14 / CRAFTSHARED INFRASTRUCTURE & HONEST TOOLING

Leave the suite better than you found it

Strict-mode locator bug in a shared step

An unscoped role lookup broke every consumer under Playwright strict mode.

FIX — scoped the locator to the searched delivery row + .first(); every check-in journey downstream got stabler for free.

Copy-pasted test-data setup

Each journey re-implemented its own cross-phase data bootstrapping.

EXTRACT — one reusable loadOrGenerateFeatureData(...) helper; plus a parseCarrier() regex for the “Name (SCAC)” contract.

A per-run config edit ritual

Every local run began with hand-editing the committed env file — and hoping you remembered to revert.

KILL IT — a gitignored .env.testing.local override; the recurring sed dance is gone for the whole team.

A filter modal rendering zero options

Mid-migration, the app itself misbehaved — the easy path was to loosen the test.

FILE IT — logged a genuine app defect instead of masking it. The oracle's “both fail → park & file” branch, used for real.

Tooling that could overclaim

A dry run that skips reconciliation could quietly look like a proven green.

REFUSE — the lane prints “RECONCILE NOT RUN — green unproven” and distinguishes executed from proven. The tool is as calibrated as the docs.
